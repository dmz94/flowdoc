Which pages work well with Decant?

Decant works best on pages written mainly to be read -- articles, blog posts, and clear information pages. If a page is mostly there to help you shop, search, fill out forms, or check live updates, it is probably not a good fit. The examples below show the difference.

If the page is mainly something you read from top to bottom, Decant will usually work well. If it is mainly something you use, browse, or interact with, it probably will not.

Good fit

Article Yale Environment 360 -- Cape Town's Baboon Wars

A long article with a clear beginning, middle, and end, written to be read straight through.

Blog post Marco Zehe -- My Journey to Ghost

A personal blog post with continuous text, headings, and a single main topic.

Information page NHS -- Mindfulness

An information page made up mostly of clear explanatory text that people read from top to bottom.

Not a good fit

These kinds of pages usually do not convert well.

Dashboard ESPN -- Pittsburgh Steelers

A live team page built around scores, schedules, and stats, not one main piece of reading.

Shopping page Amazon

A page for browsing and buying products, with many choices and moving parts instead of one main article.

Directions tool Google Maps

A tool for searching places and getting directions, not a page meant to be read like an article.
